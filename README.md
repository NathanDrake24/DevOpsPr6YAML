# DevOpsPr5YAML

Отчет по выполнению практического задания по YAML
Цель задания
Создание YAML-файла, демонстрирующего все основные типы данных формата YAML с последующей конвертацией в JSON для проверки корректности.

Выполненные шаги
1. Создан YAML-файл со всеми типами данных

<img width="1000" height="380" alt="image" src="https://github.com/user-attachments/assets/b2d0b42f-d1f6-4215-a4da-ec4decaeb1e4" />

Входной YAML-файл содержит следующие типы данных:

Строки и числа (строки 2-3):

yaml
name: "App"        # Строковый тип
port: 3000         # Числовой тип
Логические значения (строка 6):

yaml
enabled: true      # Логический тип (boolean)
Списки (массивы) (строки 8-10):

yaml
tags:              # Список элементов
- dev
- test
Словари (Map/объекты) (строки 15-16):

yaml
config:            # Словарь (ключ-значение)
  key: value
Перечисления (строка 13):

yaml
mode: production   # Строка как перечисление
Якоря и ссылки (Anchor & Alias) (строки 19-23):

yaml
common: &anchor     # Якорь (определение шаблона)
  timeout: 10

servicel: *anchor   # Ссылка на якорь
service2: *anchor   # Ссылка на якорь
2. Конвертация YAML в JSON
YAML-файл успешно сконвертирован в JSON формат, что подтверждает корректность синтаксиса и структуры:

json
{
  "name": "App",
  "port": 3000,
  "enabled": true,
  "tags": ["dev", "test"],
  "mode": "production",
  "config": {"key": "value"},
  "common": {"timeout": 10},
  "servicel": {"timeout": 10},
  "service2": {"timeout": 10}
}
Проверка корректности
✅ Все типы данных сохранены правильно:
Строки → "App", "production", "value"

Числа → 3000, 10

Логические значения → true

Списки → ["dev", "test"]

Словари → вложенные объекты config, common, etc.

Якоря → корректное копирование структуры {"timeout": 10}

✅ Особенности реализации:
Якоря: Использование &anchor для определения шаблона и *anchor для повторного использования

Вложенность: Корректное сохранение иерархической структуры

Типизация: Автоматическое определение типов данных при конвертации
